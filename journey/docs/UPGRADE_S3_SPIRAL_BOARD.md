## 2. 设计方案：矩形内旋螺旋（蜗牛盘）

72 格沿一条**矩形螺旋**由外向内盘旋，**第 1 格在最外圈、第 72 格（终点）在正中心**。

- 布局：3 圈左右的矩形螺旋，臂与臂之间留 1 格间距（看得出一圈圈的轨道），顺时针内旋。
- 几何（已用算法验证，见预览图）：
  - 设计分辨率 1920×1080；螺旋居中，中心 (960,540) = 第 72 格。
  - 第 1 格在外圈右上（约 0.766, 0.162），顺时针向内。
  - 相邻格间距均匀（约 75px），转角是干净的 90°。
- **优点**：规整、对称、终点居中视觉聚焦强；"层层逼近灵山"的取经隐喻天然契合。

## 3. 逻辑影响评估（重要）

**好消息：游戏逻辑几乎不用动。** 72 格的 index 顺序、事件、技能、判胜全部按 index 走，与坐标形状无关：

- `board_72.json`（区域/类型/事件）**不变**。
- `BoardManager` 坐标来源仍是 `board_path.json`（S1 已是这套机制），**只换点位数据**，代码零改动或极小改动。
- 移动 `get_move_path`、四角同格、碰撞——都按 index，与螺旋无关，**不受影响**。

**唯一要想清楚的：8 个西游区域怎么落到螺旋上。**

现状区域是线性分段（花果山 1–9 → … → 灵山 64–72）。螺旋上，index 仍连续，所以：
- 外圈 = 花果山/高老庄（前段），越往内 = 越接近灵山（64–72 在最内圈+中心）。**叙事上「由外向内、层层向西天」反而更顺**。
- 但螺旋的「一条臂」可能横跨两个区域的分界，美术上区域地貌的过渡不像直线那么自然。

→ 见 §5 待确认点。

## 5. 待确认点

1. **螺旋圈数/形状**：默认 **3 圈矩形内旋、终点正中心**（见预览图）。可选更多/更少圈数，或圆角更圆。接受默认？
2. **8 区域如何铺到螺旋**（核心）：
   - 方案 (i)：**按 index 段分配**——外圈→内圈依次是花果山…灵山，灵山在最内圈+中心终点。地貌随螺旋一圈圈过渡（推荐，叙事顺）。
   - 方案 (ii)：**按地图象限分区**——四个角/区域各放一个地貌主题，不严格跟 index。视觉更分明但和 index 脱节。
   - 推荐 **(i)**。
3. **中心终点视觉**：第 72 格中心要不要做成醒目的「灵山/大雷音寺」终点台（比普通格大、有光效）？建议要。
4. **路面格位形态**：沿用 S1 的统一石板凹槽？接受默认。
5. **是否保留 S1 的 board_map_v0**：S3 会再覆盖一次 board_map.png，建议把当前 S1 版另存为 `board_map_s1.png` 备份。
