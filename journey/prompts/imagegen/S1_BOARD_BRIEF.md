在现有西游取经路背景的基础上，**沿那条蜿蜒的路，从花果山到西天灵山，均匀画出 72 个清晰的格位**（像大富翁/飞行棋的路面格子），并给出每个格位的中心坐标。

## 3. 72 格位的画法要求（最关键）

1. **数量正好 72 个**，沿路**均匀分布**（相邻格间距尽量一致，视觉上像等距路标）。
2. 每个格位是一个**清晰、独立、居中于路面**的落点——形态二选一（见 §6 待确认，默认用 (a)）：
   - (a) **路面石板/凹槽圆盘**：浅色石板镶在路上，棋子站上去。
   - (b) **莲台/光点**：发光圆点。
3. 格位**不要画事件图标/箭头/颜色区分**——格子类型由编码侧叠加现有 6 类图标（奖励/惩罚/驿站等）渲染在格位上。你只画**统一的空格位**。
4. 格位大小：直径约占图宽的 **3.5%–4.5%**（≈ 70–86px @1920 宽），保证缩小后仍可辨、且能放下一个棋子。
5. 顺序连续：沿路从格1到格72**单调前进不回头**、不交叉打结（路可弯，但走向不歧义）。
6. 起点格1、终点格72 必须在路的**两端**，对应区域内。

---

## 4. 坐标表契约 `data/board_path.json`（必须严格遵守）

编码侧靠这张表把棋子落到格位上，格式错一点就对不上。

- **坐标系**：原点在图**左上角**，**x 向右、y 向下**。
- **归一化**：x = 像素x / 1920，y = 像素y / 1080，**取值 ∈ [0.0, 1.0]**，保留 **4 位小数**。
- **点 = 格位中心**：第 i 个点必须是图里第 i 个格位圆盘的几何中心。
- **数组长度正好 72**，index 从 **1 到 72 连续递增**。

格式（严格）：

```json
[
  { "index": 1,  "x": 0.0700, "y": 0.8200 },
  { "index": 2,  "x": 0.1050, "y": 0.7600 },
  ...
  { "index": 72, "x": 0.9200, "y": 0.2600 }
]
```

> 字段固定为 `index` / `x` / `y`，`snake_case`，无多余字段。
> 文件放 `data/board_path.json`（这是编码侧约定路径，Codex 按此命名产出）。
