# S2 任务清单：角色选择 + 结算 两张全屏背景（Codex + imagegen）

> 拥有方：**Codex + imagegen**。第二阶段优化「为开始-角色选择、结束-结算做美术背景」。
> 关联：风格基线 [`ART_ASSET_PROMPTS.md`](./ART_ASSET_PROMPTS.md) §1、对局背景 [`S1_BOARD_BRIEF.md`](./S1_BOARD_BRIEF.md)（保持风格统一）。
> 目标：两张 **1920×1080** 全屏背景，**为 UI 预留好空白区**，使文字/卡片/按钮叠上去清晰不被画面元素干扰。

---

## 0. 全局约定（两张图通用）

- **尺寸**：1920×1080（16:9），RGB，**不透明背景图**（区别于角色/图标的透明 PNG）。
- **风格**：严格沿用 §1 基线 + 与 S1 对局背景 `board_map.png` **同一世界观/笔触**——Q 版西游、圆润明亮、童话卡通、不写实不恐怖、**无图内文字**。
- **坐标系**：原点左上，x 向右、y 向下，单位像素，基于 1920×1080。下文「预留区」均给绝对像素矩形。
- **预留区含义**：该矩形内**画面要简洁、低对比、不放主体/强细节/高频纹理**，以便叠加 UI 文字与控件仍清晰可读。预留区可以有柔和的底色/虚化光晕/半透明卡牌槽，但**不要放抢眼的角色、建筑、文字**。
- **安全边**：四周留 ≥40px 不放关键信息（投影仪/不同比例屏幕可能轻微裁切）。

## 1. 图一：角色选择背景 `select_bg.png`

**用途**：玩家进入游戏后选 4 个取经人之一的界面背景。

**文件**：`assets/backgrounds/select_bg.png`

**画面建议**：取经队伍出发前的场景——花果山/长安城门/取经启程的祥和清晨。氛围积极、邀请感。可在**左右两侧或上下边缘**放景物（云、山、旗、桃树），**中部大片留给 UI**。

**必须预留的 UI 空间**（这些区域请保持简洁低对比）：

| UI 元素 | 预留矩形 (x, y, w, h) | 说明 |
|---|---|---|
| 标题「选择你的取经人」 | (360, 60, 1200, 90) | 顶部居中横幅，背景宜深色留白或柔光带，保证白字可读 |
| 4 张角色卡片区 | (260, 360, 1400, 420) | 横排 4 张卡片（每张约 320×420，间距 ~40）。**此横带务必干净**——卡片是不透明的，但卡片之间会露出背景，避免杂乱 |
| （可选）卡片槽位提示 | 同上区域内 4 个等距槽 | 若想画 4 个柔和的「卡牌底座/光台」呼应卡片位置，中心 x ≈ 480 / 800 / 1120 / 1440，y ≈ 570，直径 ≤ 360；纯装饰、低对比 |

> 4 张卡片中心 x 坐标（编码侧布局基准）：**480 / 800 / 1120 / 1440**，垂直中心 y ≈ **570**。
> 卡片本身（头像+名字+技能说明+按钮）由代码绘制，背景只需保证这条横带干净 + 可选的 4 个装饰光台对齐这 4 个 x。

## 2. 图二：结算背景 `result_bg.png`

**用途**：分出胜负后的结算界面背景（显示胜者立绘 + 四方排名 + 重新开始按钮）。

**文件**：`assets/backgrounds/result_bg.png`

**画面建议**：抵达西天灵山的庆典/功德圆满场景——金色祥云、莲台、灵山佛光、漫天花雨。氛围**庆祝、辉煌但不刺眼**（注意预留区低对比）。中部偏上给胜者立绘、中部给排名、下方给按钮。

**必须预留的 UI 空间**：

| UI 元素 | 预留矩形 (x, y, w, h) | 说明 |
|---|---|---|
| 胜者横幅文字 | (360, 90, 1200, 80) | 顶部「🏆 XX 抵达西天，取经成功！」白字，背景宜柔光留白 |
| 胜者立绘区 | (810, 200, 300, 300) | 居中偏上，放胜者头像（代码绘制）。此圆区背景宜柔和、可有光环但别太满 |
| 排名列表区 | (730, 520, 460, 280) | 居中四行排名文字（代码绘制）。**务必干净低对比**，保证多行文字清晰 |
| 重新开始按钮 | (860, 830, 200, 60) | 底部居中按钮（代码绘制） |

> 关键留白带：**水平 x∈[700,1220]、垂直 y∈[180,900] 的中央纵向通道**要整体偏简洁——胜者立绘、排名、按钮都在这条通道上。两侧 (x<700 / x>1220) 可放祥云莲台佛光等庆典景物。
