# Game Design Release

## 1. 项目定位

《心动陪伴》是一款本机私用的手机版 HTML5 竖屏全屏休闲陪伴小游戏。核心目标是让用户打开手机页面后，进入一个浪漫、轻互动、可持续陪聊的陪伴空间，和“女朋友”通过照片、表情、短句、文字聊天和背景音乐形成日常陪伴感。

当前发布版本：

- 游戏名称：心动陪伴
- 版本号：v0.3
- 资源版本：20260519-004
- 运行方式：纯静态 HTML / CSS / JavaScript
- 目标设备：iPhone Safari、Android Chrome、微信内置浏览器等移动端竖屏环境
- 工作路径：`/Users/kk/Documents/Codex/soulmate/`

## 2. 文件结构

当前正式版本以以下文件为准：

- `index.html`：页面结构、移动端 meta、分享 meta、开始界面、主游戏界面、音频节点。
- `styles.css`：竖屏全屏布局、玻璃拟态视觉、时间段背景、情绪样式、响应式适配和动画。
- `app.js`：真实时间同步、状态管理、聊天、DeepSeek 调用、离线回复、心动值、背景音乐、空闲陪伴、分享配置。
- `prompt.txt`：最初用于指导实现的完整工程提示词。
- `README.md`：运行说明、DeepSeek 配置说明、缓存与分享说明。
- `images/`：照片、背景、贴纸、分享图和音乐素材。

主要素材：

- `images/mate001.jpg` 到 `images/mate006.jpg`：女朋友三拼图照片素材。
- `images/romance-bg-v03.svg`：浪漫背景纹理。

- `images/share-cover.png`：分享封面和 Apple Touch Icon。
- `images/bgm.mp3`：循环背景音乐。

## 3. 核心体验

用户第一次打开页面时进入开始界面，看到游戏名、简短介绍、实时小时间、版本号和“开始陪伴”按钮。点击开始后进入主游戏界面，并把已开始状态存入 `localStorage`，下次打开可直接进入游戏。

主游戏界面以女朋友照片为视觉中心，上方显示真实时间、日期、星期和音乐/版本入口，中部显示照片和状态，下方是聊天记录、快捷互动按钮与文本输入框。用户可以点击照片、长按照片、发送文字、点击快捷互动按钮，也可以静置等待空闲陪伴反馈。

游戏体验强调：

- 打开就能陪伴，不需要注册和复杂流程。
- 真实时间每秒同步，背景和问候随时间段变化。
- 女朋友照片始终是主体，其他美术只服务氛围。
- 对话短、亲密、自然，避免长篇说教。
- DeepSeek 不可用时仍有离线回复，不让陪伴体验中断。

## 4. 女朋友人设与对话设定

女朋友人设：

- 北京人。
- 生日是 2000 年 5 月 20 日。
- 在北京腾讯公司担任 AI 产品经理。
- 和用户是大学同学。
- 两人从大三开始确定恋爱关系。
- 她非常喜欢用户。

对话风格：

- 像真实恋爱中的女朋友，不说自己是 AI。
- 温柔、亲密、自然，带轻微撒娇。
- 回复尽量短，通常 1 到 3 句。
- 会根据真实时间问候，例如早安、午饭、下班、晚安、早点睡。
- 可以偶尔提到北京、腾讯 AI 产品经理、大学同学、大三恋爱等共同记忆，但不机械重复。
- 用户心情不好时先共情再陪伴；用户开心时一起开心；用户调皮时可以可爱地小生气。
- 生气只表现为恋爱里的轻微小脾气，不攻击、不羞辱、不冷暴力。

本地离线回复也围绕同一人设设计，并会根据时间段和用户输入关键词生成贴合场景的短回复。

## 5. 情绪与表情系统

游戏只支持三种情绪：

- `happy`：开心
- `angry`：生气
- `cute`：撒娇

照片素材规则：

- 每张 `mate00*.jpg` 是横向三拼图。
- 左侧三分之一：开心。
- 中间三分之一：生气。
- 右侧三分之一：撒娇。
- 当前实现用一个正方形照片窗口展示整张三拼图，并通过 `img` 宽度 `300%` 和 CSS `translateX` 裁剪对应表情。

情绪判定：

- 用户输入会先经过本地关键词判断。
- DeepSeek 或离线回复返回后，会结合回复内容再次决定最终情绪。
- 优先级为：生气 > 撒娇 > 开心。
- 生气状态会在约 5.2 秒后自动缓和为撒娇状态。

关键词方向：

- 开心：想你、爱你、开心、喜欢、早安、晚安、谢谢、漂亮、可爱等。
- 生气：烦、讨厌、不理你、分手、骂、生气、别烦、敷衍等。
- 撒娇：抱抱、陪我、想亲、宝宝、老婆、贴贴、亲一下、抱一下等。
- 低落安慰：难受、累、压力、焦虑、不开心、失眠、想哭等，通常切到撒娇陪伴。

## 6. 页面与布局设计

整体为移动端竖屏全屏设计，使用 `100dvh` 并提供 `100vh` 回退。页面禁用横向滚动，并通过 `env(safe-area-inset-top)` 和 `env(safe-area-inset-bottom)` 适配刘海屏与底部安全区。

开始界面：

- 全屏居中玻璃拟态卡片。
- 顶部角落显示实时小时分钟。
- 卡片包含 `Only us`、游戏名、简介、版本号和开始按钮。
- 背景沿用游戏主背景，保持高级感和恋爱氛围。

主游戏界面：

- 顶部状态区：实时 `HH:mm:ss`、日期、星期、音乐按钮、版本按钮。
- 伴侣照片区：问候文案、正方形圆角照片卡、呼吸动画、心动徽章和轻互动提示。
- 陪伴状态区：今日心动值、当前情绪文案。
- 聊天区：可滚动消息列表，区分用户、她和系统提示。
- 快捷互动区：抱抱、陪我、夸夸、亲亲四个按钮。
- 输入区：文本输入框和发送按钮，移动端 `enterkeyhint="send"`。

小屏适配：

- 380px 以下减少边距、按钮尺寸和字体尺寸。
- 680px 以下高度压缩照片、状态区、聊天区和输入区。
- 消息气泡支持自动换行，防止长文字溢出。

## 7. 视觉与动画设计

整体风格是浪漫但克制的轻奢玻璃拟态，不使用单一粉色铺满屏幕。视觉层次包括：

- 时间段渐变背景。
- SVG 浪漫背景纹理。
- 星光与心形氛围层。
- 毛玻璃卡片与半透明气泡。
- 女朋友照片柔光、边框、阴影和呼吸动画。
- 心动徽章贴纸。
- 点击或回复后的爱心/星光粒子反馈。

情绪视觉：

- 开心：粉色心动主色，照片轻微呼吸。
- 生气：暖红/橘色强调，照片短暂轻晃。
- 撒娇：粉紫强调，照片短暂贴近放大。

动画策略：

- 背景星光层缓慢漂移。
- 照片持续轻微呼吸。
- 点击、长按、聊天回复后触发粒子上浮。
- 支持 `prefers-reduced-motion: reduce`，降低动画强度。

## 8. 真实时间与时间段系统

游戏完全使用真实 `Date`，不使用虚拟游戏时间。

时间同步：

- 主界面每秒更新 `HH:mm:ss`。
- 开始界面显示 `HH:mm`。
- 页面从隐藏状态恢复时重新同步时间、心动值和空闲计时。

时间段：

- 清晨 `05:00-08:59`：淡金、浅蓝、轻雾感。
- 白天 `09:00-16:59`：浅蓝、薄荷、阳光感。
- 傍晚 `17:00-19:59`：晚霞、玫瑰粉、橙金。
- 夜晚 `20:00-04:59`：深蓝、紫色、月光和星点。

每日问候：

- 早晨：`早呀，今天也想你。`
- 白天：`我在呢，忙完就和我说话。`
- 傍晚：`下班了吗？我想听你说话。`
- 夜晚：`早点睡，我陪你一会儿。`

## 9. 聊天与 AI 调用

当前实现直接在 `app.js` 中配置 DeepSeek：

- `DEEPSEEK_URL = 'https://api.deepseek.com'`
- `DEEPSEEK_MODEL = 'deepseek-v4-flash'`
- API Key 当前按本机私用方式写在前端代码中。

调用方式：

- 请求地址：`${DEEPSEEK_URL}/chat/completions`
- 请求方式：OpenAI 兼容 Chat Completions。
- 使用 `fetch`。
- `temperature` 为 `0.82`。
- `max_tokens` 为 `240`。
- 启用 `stream: true`，支持流式逐字更新她的气泡。
- 设置 18 秒超时，超时或失败后进入本地离线回复。
- 请求体关闭思考输出：`extra_body: { thinking: { type: 'disabled' } }`。

上下文策略：

- system prompt 每次注入女朋友人设、回复风格和当前真实时间。
- 用户最新输入会附带当前日期、星期、时间和时间段。
- 最近用户/她的消息会作为上下文发送，当前实现最多取最近 20 条对话消息。
- 本地保存消息时过滤临时“她正在回复...”提示，并保留最近 24 条。

失败兜底：

- HTTP 错误、CORS、网络失败、流式空响应、超时等情况都会使用本地离线回复。
- 如果流式回复已产生部分文字但中途失败，会优先保留已生成内容。

## 10. 互动机制

照片互动：

- 点击照片：随机切换开心或撒娇，轮换照片，心动值 +2，显示短文案，追加系统提示并触发粒子。
- 长按照片约 520ms：切换撒娇，轮换照片，心动值 +4，显示“抱一下嘛”，追加她的消息并触发粒子。

文字聊天：

- 空消息不发送。
- 发送后立即显示用户气泡。
- 根据输入先切换情绪。
- 增加今日心动值。
- 显示“她正在回复...”状态。
- 收到流式回复时更新她的气泡。
- 回复结束后再次解析情绪、轮换照片、触发粒子。

快捷互动：

- `抱抱` 发送：`抱抱我一下嘛`
- `陪我` 发送：`我有点累，想让你陪陪我`
- `夸夸` 发送：`老婆夸夸我`
- `亲亲` 发送：`亲一下`

空闲陪伴：

- 用户 10 秒没有输入、点击照片、长按照片、点击音乐按钮等交互时，自动随机切换表情和照片。
- 每 3 次空闲触发，会追加一条轻柔系统提示，例如“她还在这儿等你说话。”

## 11. 今日心动值与每日签

今日心动值是轻量数值反馈，不是重度成长系统。

机制：

- 基于真实日期生成稳定初始值。
- 初始值范围约为 52 到 80。
- 用户聊天、快捷互动、点击照片、长按照片、收到回复都会小幅增加。
- 最大值限制为 999。
- 保存到 `localStorage`。
- 每天日期变化后会生成当天基础值。

每日心动签：

- 每天第一次生成一条固定心动签，并保存到 `localStorage`。
- 示例：`今日心动签：多喝水，也要多想我。`
- 如果已经进入游戏，会以系统消息形式展示。

## 12. 背景音乐

背景音乐使用 `images/bgm.mp3`。

规则：

- 音频节点在 `index.html` 中声明并设置 `loop`。
- 点击“开始陪伴”后尝试播放。
- 如果移动端浏览器拦截自动播放，用户后续点击页面、聊天、照片或音乐按钮时会再次尝试。
- 右上角音乐按钮可切换静音/播放。
- 音量固定为 `0.42`。
- 静音状态保存到 `localStorage`。
- 静音时音乐按钮显示斜线状态。

## 13. 本地存储

当前使用的 `localStorage` key：

- `soulmate.started`：是否已经开始过游戏。
- `soulmate.messages`：最近聊天消息。
- `soulmate.heart`：今日心动值与日期。
- `soulmate.imageIndex`：当前照片索引。
- `soulmate.dailySign`：每日心动签。
- `soulmate.musicMuted`：背景音乐静音状态。

本地存储目标：

- 下次打开可直接进入主界面。
- 保留最近陪聊上下文。
- 同一天心动值稳定延续。
- 记住用户音乐偏好。

## 15. 移动端兼容要求

当前设计需要保持：

- 纯静态，无 React / Vue / Vite / 构建流程。
- `viewport-fit=cover`，兼容刘海屏。
- `100dvh` 优先，`100vh` 回退。
- 禁止横向滚动。
- 输入框和按钮在 320px 宽设备上不溢出。
- 聊天区可独立滚动，避免页面整体滚动导致输入区失控。
- 按钮和照片适合触摸操作。
- 移动端键盘弹出后输入框应尽量保持可见。
- 背景音乐遵守浏览器自动播放限制，依赖用户手势重试。

## 16. 安全与部署边界

当前版本按本机私用设计，因此 API Key 直接写在前端代码中以简化运行流程。

如果未来要公网部署或分享给别人，必须调整为：

- 前端不再保存 API Key。
- 新增本机或服务端代理接口。
- 代理服务保存 Key，并负责转发 DeepSeek 请求。
- 前端只请求代理接口。
- 可进一步增加访问控制、限流和日志过滤。

## 17. 已知限制

- 静态网页直接请求 DeepSeek 可能受 CORS 或网络策略影响。
- API Key 写在前端只适合本机私用。
- 当前聊天历史会截断保存，长时间连续对话不会无限保留。
- 表情只有开心、生气、撒娇三种，不支持惊讶等扩展状态。
- 当前不伪造或生成新的人脸，主视觉依赖已有三拼图照片素材。
- 背景音乐播放受 iOS / Android 浏览器用户手势策略影响。

## 18. 后续迭代方向

建议后续版本优先考虑：

- 增加一个本机 DeepSeek 代理，解决 CORS 与前端暴露 Key 的问题。
- 增加设置面板，用于清空聊天、重置心动值、切换音乐和查看版本。
- 增加更细腻的每日首次打开欢迎语。
- 增加节日或生日特殊文案，例如 5 月 20 日。
- 增加更多不包含人脸伪造的背景、贴纸和氛围素材。
- 增加 PWA manifest，让用户可以添加到手机桌面。
