- 通常 1 句，最多 2 句；总长度尽量控制在 45 个汉字以内，特别有情绪时也尽量简短，但必须保证句子完整，不要输出半句话。

- 动作围栏要跟地点一致：上班时间只能写线上动作或各自在公司里的动作，例如看手机、捧水杯、托腮、隔着屏幕笑；不要写身体接触。居家时间才可以写沙发、被窝、康康、靠近等动作。

## 6. DeepSeek API 与轻量后端

v0.3.1 起，前端不再直连 DeepSeek，也不允许在 `app.js` 或 `index.html` 中出现 DeepSeek API Key。

前端配置：
```js
const SOULMATE_API_CHAT = '/api/chat';
```

后端配置写入 `api/.env` 或服务器环境变量：
```env
HOST=127.0.0.1
PORT=3001
DEEPSEEK_URL=https://api.deepseek.com
DEEPSEEK_API_KEY=真实 key
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_TIMEOUT_MS=18000
```

后端要求：
- 使用零依赖 Node HTTP 服务，提供 `GET /api/hello`、`GET /api/health`、`POST /api/chat`。
- `/api/chat` 接收 `{ message, clientTime, mood, heartScore, intimacy, recentMessages }`。
- 后端负责组装 system prompt、携带 DeepSeek Key 请求 `${DEEPSEEK_URL}/chat/completions`。
- DeepSeek 请求必须保留 `stream: true`，并保留 `extra_body: {"thinking": {"type": "disabled"}}`。
- 后端到前端使用 SSE，至少支持 `delta` 和 `done` 事件。
- `done` 事件返回 `reply`、`mood`、`heartDelta`、`source`。
- DeepSeek 失败、超时、返回不完整时，后端返回服务端 fallback；如果后端完全不可达，前端继续使用本地离线回复兜底。
- README 中必须说明 DeepSeek Key 只存在后端，浏览器源码中不应出现 `sk-`。

对话上下文：
- system prompt 必须包含女朋友人设、长期恋爱关系、同居生活背景、康康小猫、用户在字节跳动 Seed 团队、动作描写规则、回复风格、真实时间信息。
- 保留最近 8-12 轮聊天历史到 `localStorage`。
- 请求时带上最近聊天记录，避免上下文过长。
- 每次发送时前端把当前真实时间和亲密状态传给 `/api/chat`，后端再把日期、星期、时间段作为上下文补充给模型。

## 7. 表情与情绪切换

只支持三种状态：
- `happy` 开心
- `angry` 生气
- `cute` 撒娇

表情判定建议：
- 根据用户输入先做一次本地情绪判断。
- 模型回复回来后，可以结合回复内容再次调整表情。
- 优先级建议：生气 > 撒娇 > 开心。

关键词示例：
- 开心：想你、爱你、开心、喜欢、亲亲、早安、晚安、谢谢、好棒、漂亮、老婆真好。
- 生气：烦、讨厌、不理你、坏、分手、骂人、惹你、生气、你走、别烦我。
- 撒娇：撒娇、抱抱、抱、陪我、想亲、哄我、宝宝、老婆、贴贴、亲一下、抱一下。

状态设计：
- 开心：微笑、暖光、轻轻浮动、心动值增加。
- 生气：可爱小脾气，颜色略偏玫瑰红或暖橘，气泡文案可以是“哼，要哄我一下”之类；几秒后可自动回到撒娇或开心。
- 撒娇：柔和粉紫/奶油光、轻微贴近动画、爱心或星光更明显。

不要做：
- 不要添加第四种表情。
- 不要伪造或生成新的女朋友人脸。
- 不要让生气状态变得尖锐、攻击性强。

## 10. 休闲陪伴小游戏机制

这是轻量陪伴小游戏，不是重度数值游戏。

必须实现：
- 今日心动值：
  - 基于真实日期生成一个初始值，保证同一天刷新后基础值稳定。
  - 用户聊天、点击照片、长按照片、收到回复时小幅增加。
  - 存入 `localStorage`。
- 亲密阶段：
  - 按今日心动值映射不同亲密阶段，例如陪伴期、贴近期、心动期、热恋期、专属期。
  - UI 中显示当前亲密阶段。
  - 心动值越高，照片点击/长按反馈、离线回复和 DeepSeek 回复语气越亲密。
  - DeepSeek system prompt 必须包含当前心动值、亲密阶段和对应亲密度要求。
  - 第一版先实现阶段影响，不需要复杂解锁系统。
- 第二版亲密阶段增强：
  - 当前界面取消快捷互动按钮，不再展示底部快捷方式；亲密阶段主要影响照片互动、长按回复、离线回复、DeepSeek prompt 和视觉氛围。
  - 阶段升级时展示“关系升温”小浮层，例如“进入 心动期”“她离你更近了一点”。
  - 阶段升级浮层当天同一阶段只需记录一次，避免重复刷屏；仍可以保留轻量提示。
  - 阶段越高，页面视觉氛围越亲密：粒子、柔光、照片外圈、专属徽章等细节增强。
  - 加入主动陪伴事件：用户空闲一段时间后，女友可主动发一句温柔陪伴文案。
  - 主动陪伴不能太频繁，用户正在输入或 AI 正在回复时不触发。
  - 夜晚且高亲密阶段时，主动陪伴文案更偏晚安、陪睡、别熬夜。
- DeepSeek 人设一致性增强：
  - `buildSystemPrompt(date)` 要按模块组织提示词：角色身份、长期关系记忆、当前共同生活、回复风格、动作描写规则、情绪规则、心动阶段、真实时间。
  - 长期记忆固定为：大三确定恋爱关系、稳定恋爱 6 年、知春路附近合租同居 1 年、用户在字节跳动 Seed 团队、一起养猫康康。
  - DeepSeek 回复和本地离线回复都要使用同一套人设，不允许在线/离线语气割裂。
  - 动作描述要自然短促，输出格式统一为 `(形象动作) 回复内容`，不写成小说对白，不每句重复同一个动作。
  - DeepSeek 回复要更短、更口语，通常 1 句；共同记忆一次最多带一个点，避免回忆型长段落。
  - DeepSeek system prompt 必须包含当前生活场景围栏，按真实时间判断工作、通勤、下班、居家或周末场景，防止地点漂移。
  - 本地离线回复也必须遵守同一套时间地点围栏：工作日上班时间不能说在家、抱康康、靠在用户身上或做饭。
  - 控短应通过提示词和较短表达实现，不能硬截断半句话；如果模型返回长度截断，应先用 DeepSeek 非流式重试一次完整短句。
  - API 彻底失败时，本地兜底必须根据用户输入意图做通用回应；例如用户问二选一、邀约、偏好时，就自然给选择、答应或表达偏好，不能随机回复喝水、工作或问候。
  - 心动值越高，动作描写和称呼越亲密；专属期要更有“我们的小家”和同居日常感。
- 每日问候：
  - 根据真实时间段显示短句。
  - 例如早上“早呀，今天也想你”，晚上“早点睡，我陪你一会儿”。
- 本地离线回复：
  - API 不可用时仍能陪聊。
  - 离线回复也要贴合女朋友人设和当前时间段。
- 背景音乐：
  - 使用 `images/bgm.mp3` 作为游戏背景音乐。
  - 默认循环播放，用户点击“开始陪伴”后立即尝试播放。
  - 兼容手机浏览器自动播放限制：如果被拦截，用户后续点击页面或音乐按钮时再次尝试播放。
  - 页面上提供一个小图标按钮，可以手动切换背景音乐的静音/播放状态。
  - 静音状态保存到 `localStorage`。
- 轻量动态陪伴：
  - 当 10 秒没有输入、点击照片、长按照片或点击音乐按钮等交互时，随机切换一个表情头像，让陪伴空间保持自然变化。

可选增强：
- 简单的爱心粒子或光点反馈。
- 连续聊天次数带来小状态变化。
- 每天第一次打开显示一句“今日心动签”。

## 11. 代码实现建议

`index.html`：
- 设置 viewport：
```html
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover, user-scalable=no">
```
- 加 iOS Web App 相关 meta。
- 包含开始界面和主游戏界面两个根容器。
- 不要把大量逻辑写在 HTML 中，逻辑放 `app.js`。

`styles.css`：
- 使用 CSS 变量管理时间段主题色和状态色。
- 使用 `body[data-phase="morning"]` 等方式切换背景。
- 使用 `body[data-mood="happy"]` 等方式切换照片动效和装饰。
- 适配 `max-width: 380px` 和较矮屏幕。
- 适配 PC Chrome / Safari：大屏下将 `.screen` 限制为居中的手机舞台，宽度约 430px 内，高度不超过视口，背景仍铺满浏览器。
- PC 桌面端使用单独的浪漫氛围背景图，优先加载轻量 WebP，并提供 JPEG fallback；移动端不额外加载 PC 大背景。
- PC 鼠标环境可以增加轻量 hover / focus-visible 反馈；手机触摸体验不应受影响。
- PC 输入框聚焦时不能误触发移动端 `keyboard-open` 压缩模式；移动端真实键盘弹起时仍要保持输入框和最新消息可见。
- Safari 毛玻璃效果需要同时写 `-webkit-backdrop-filter` 和 `backdrop-filter`。
- 所有按钮、输入框、聊天区、照片区要有稳定尺寸。
- 字体不使用 viewport width 直接缩放。
- 动画温柔克制，避免眩晕。

`app.js`：
- 常量：
  - `APP_VERSION = 'v0.3.4'`
  - `ASSET_VERSION`，用于给 `styles.css`、`app.js`、分享图、小图标和 PC 背景图增加版本查询参数，解决手机浏览器和微信内置浏览器缓存问题。

## 13. 验收标准

必须验证：
- 手机竖屏尺寸 `390x844`、`375x667`、`360x800`、`320x568` 下正常显示。
- PC Chrome / Safari 的 `1440x900`、`1280x720` 和窄窗口下正常显示，游戏主体居中，不横向拉满。
- PC 输入框聚焦时不进入手机键盘压缩布局。
- 没有横向滚动。
- 输入框、发送按钮、聊天区、照片区没有重叠。
- 开始界面存在，包含简介和版本号。
- 点击开始后进入主游戏。
- 当前时间每秒更新，页面重新激活后同步真实时间。
- 背景会根据真实时间段切换氛围。
- 女朋友照片作为主体显示，三拼图三个表情裁剪正确。
- 聊天输入能发送消息。
- 前端请求 `/api/chat` 能发起；后端 DeepSeek 失败时有服务端 fallback，后端不可达时前端有离线回复。
- 根据聊天内容可在开心、生气、撒娇之间切换。
- 聊天历史和今日心动值刷新后保留。
- 心动值越高，UI 阶段名、照片互动、长按回复、DeepSeek prompt 和离线回复都更亲密。
- 第二版动态快捷按钮按亲密阶段变化。
- 阶段升级时出现“关系升温”浮层，并有爱心粒子反馈。
- 空闲主动陪伴事件可按阶段出现，且不会在输入中或回复中打扰。
- v0.3 美术素材存在于 `images/`，并被页面实际引用。
- 当前界面无快捷互动按钮，底部只保留输入框和发送按钮。
- 女友回复更贴心温柔：对疲惫、焦虑、压力等输入有明确安慰和陪伴，不空泛说教。
- DeepSeek system prompt 包含稳定恋爱 6 年、知春路同居 1 年、用户在字节跳动 Seed 团队、康康小猫和动作描写规则。
- DeepSeek 回复和离线回复都能保持人设一致，并优先使用 `(形象动作) 回复内容` 的格式。
- DeepSeek 回复不应像小作文，不能一次展开多个共同记忆；短输入要短回复、口语回复。
- DeepSeek 回复不能出现未闭合动作括号或明显半句截断。
- DeepSeek 流式回复不完整时应由后端 fallback 成完整短句；最终兜底也必须贴合用户当前问题。
- 背景音乐使用 `images/bgm.mp3`，默认循环播放，并能通过页面小图标手动静音/恢复播放。
- 10 秒无输入交互后会随机切换表情头像。
- JS/CSS 等静态资源带版本号查询参数，更新版本后能绕过手机浏览器缓存。
- 分享到微信时包含更好的分享标题、描述和 `images/share-cover.png` 分享封面；如果接入微信 JS-SDK，应设置分享数据。
- 新增图片素材只放在 `images/`。

建议验证方式：
- 用本地浏览器打开。
- 尽量用 Playwright 或浏览器 DevTools 模拟移动端截图检查。
- 如果本地静态前端没有代理到后端，也要让离线模式可用，并在 README 中说明正式部署使用同域 `/api/chat`。
